# 训练模型链路梳理

本文档整理当前 `/root/workspaces/wujie_gsq/vla` 项目里“基于已有数据训练/微调 VLA 模型”的启动命令、涉及程序、输入输出和当前实际输入数据。

当前主链路在：

```bash
/root/workspaces/wujie_gsq/vla/openpi_modified
```

核心项目是修改过的 OpenPI，当前更明确、最值得优先跑通的是 `seatbelt` 任务的 `pi0.5 + 14 维动作空间` 训练链路。

## 一句话概览

当前训练链路是：

```text
LeRobot/AnyVerse 数据集
-> 训练配置 TrainConfig
-> 计算 norm_stats.json
-> 构造 AnyverseDataset / DataLoader
-> 数据 repack / delta action / normalization / model transform
-> 加载 pi0.5 base 预训练权重
-> JAX 训练
-> 保存 checkpoint
-> 后续 serve_policy / openloop eval / 机器人推理
```

## 涉及程序及输入输出

### 1. 训练配置加载

程序：

```bash
src/openpi/training/config.py
src/openpi/training/base_cfg.py
src/openpi/configs/cfg_pi0.5_seatbelt_14_dim.py
```

输入：

```text
--config src/openpi/configs/cfg_pi0.5_seatbelt_14_dim.py
```

输出：

```text
TrainConfig 对象
```

这个对象决定：

```text
用哪个模型
用哪些数据
从哪里读 norm stats
从哪里加载初始权重
训练多少步
checkpoint 存在哪里
是否启用数据清洗/加权预处理
```

### 2. norm stats 计算

程序：

```bash
scripts/compute_norm_stats_fast.py
```

启动命令：

```bash
uv run scripts/compute_norm_stats_fast.py \
  --config-name src/openpi/configs/cfg_pi0.5_seatbelt_14_dim.py
```

输入：

```text
TrainConfig
配置里的 root_dir
配置里的 repo_id 列表
每个 repo 的 data/*.parquet
每个 repo 的 meta/info.json
state/action 字段
is_valid 字段，如果数据里存在
```

输出：

```bash
/root/workspaces/wujie_gsq/vla/openpi_modified/assets/seatbelt_14_dim_0401/20260401/norm_stats.json
```

作用：

```text
统计 observation.state 和 action 的 mean/std/q01/q99
训练时用于归一化 state/action
如果缺失，训练阶段会报 Normalization stats not found
```

当前我之前只验证过该脚本的输出路径查询可用；正式训练前仍需要实际生成 `norm_stats.json`。

### 3. 数据集构造

程序：

```bash
src/openpi/training/data_loader.py
src/openpi/training/anyverse_dataset.py
```

关键入口：

```python
create_data_loader()
create_torch_data_loader()
create_anyverse_dataset()
MultiAnyverseDataset
AnyverseDataset
```

输入：

```text
root_dir = /mnt/
repo_id = seatbelt 配置中的 281 个数据集路径
action_horizon = 50
action_sequence_keys = ("action",)
align_dim = 14
frame_attributes_preprocessors
norm_stats.json
```

输出给训练程序的是 batch：

```text
Observation
Actions
ActionsMask
```

其中 Observation 里主要包含：

```text
图像: head / right_wrist / left_wrist 等相机视角
状态: observation.state
文本: task / prompt token
可选: joint_eef_dof_mask、optimality 等
```

Actions 是长度为 50 的 action chunk。当前配置会把部分动作转为 delta action。

### 4. 数据清洗、筛选、加权

程序：

```bash
src/openpi/training/frame_attributes_preprocessors
```

当前 seatbelt 配置实际启用：

```python
VelocityBasedStaticDetector
PruneHeadTailStaticValidMaskPreprocessor
GripperCountValidMaskPreprocessor
RepoNameMatchSampleWeightPreprocessor
GripperCountSampleWeightPreprocessor
```

这些不是离线改写原数据，而是在 Dataset 初始化阶段生成：

```text
valid_mask
sample_weight
segment_id
optimality
```

当前规则大意：

```text
1. 根据关节/夹爪速度检测静止帧。
2. 普通数据去掉开头静止段，保留结尾。
3. avoid / wandering 数据会裁掉尾部静止段。
4. self_play / infer 数据跳过静态裁剪。
5. recover_2_n_move 数据在特定右夹爪 close 事件之后置为无效。
6. repo 名里包含 insert 的数据整体 2 倍采样权重。
7. insert 数据里最后一次右夹爪 open 前 4 秒再加 2 倍权重，合计可到 4 倍。
```

输出：

```text
训练时实际采样的 frame index
每个 frame 的有效性 mask
每个 frame 的采样权重
```

这一步是当前“数据优化”最核心的部分。

### 5. 数据字段转换

程序：

```bash
src/openpi/training/base_cfg.py
src/openpi/policies/gr00t_policy.py
src/openpi/transforms.py
```

当前 `Gr00tLerobotDataConfig` 会做字段重映射：

```text
observation.images.head        <- observation/front_image
observation.images.right_wrist <- observation/wrist_image
observation.images.left_wrist  <- observation/wrist_image_lf

```

之后会做：

```text

```

输出：

```text
模型可直接消费的 Observation / Actions / ActionsMask
```

### 6. 训练主程序

程序：

```bash
scripts/train.py
```

启动命令：

```bash
XLA_PYTHON_CLIENT_MEM_FRACTION=0.98 \
uv run scripts/train.py \
  --config src/openpi/configs/cfg_pi0.5_seatbelt_14_dim.py
```

输入：

```text
TrainConfig
DataLoader 输出的 batch
norm_stats.json
pi05_base 初始权重
JAX/XLA/GPU
```

训练时做：

```text
初始化模型
加载 pi05_base 权重
构造 optimizer 和 lr schedule
取 batch
计算 flow matching/action loss
按 actions_mask 过滤无效动作位置
反向传播更新参数
记录 loss、grad_norm、param_norm、learning_rate、各动作维度 loss
定期保存 checkpoint
```

输出：

```bash
checkpoints/seatbelt_14_dim_0401/seatbelt_14_dim_0401_exp/<step>/
```

目录中会保存：

```text
模型参数
训练状态
配置副本
assets/norm_stats
wandb_id.txt，如果启用 wandb
```

## 当前输入数据

当前主配置从这里读数据：

```bash
/mnt/oss_data/anyverse_human_data_record/arxx5_bimanual/seatbelt
```

配置中实际写法是：

```python
ROOT_DIR = "/mnt/"
REPO_ID = [
    "oss_data/anyverse_human_data_record/arxx5_bimanual/seatbelt/...",
    ...
]
```

因此单个数据集实际路径形如：

```bash
/mnt/oss_data/anyverse_human_data_record/arxx5_bimanual/seatbelt/seatbelt.single.hang.baichenglong.20260205.batch.5
```

当前 `cfg_pi0.5_seatbelt_14_dim.py` 纳入了 281 个 repo。

按 repo 名粗略分类：

```text
hang: 137
insert: 44
hang_move: 33
recover_2: 24
take_off: 22
recover_1: 21
avoid_obstacle: 18
hang_continuous: 6
wandering: 3
recover_0: 2
other: 13
```

这些数据大致覆盖：

```text
正常挂安全带: hang / hang_move
插入动作: insert / fix_insert
连续挂载: hang_continuous
恢复动作: recover_0 / recover_1 / recover_2 / collision_recovery
避障与异常恢复: avoid_obstacle / avoid_collision / wandering
取下/回家: take_off / back_home
```

从训练角度看，它不是单纯的成功示教数据，而是混合了：

```text
专家示教
关键子动作
恢复数据
避障数据
异常轨迹
回家/收尾动作
```

这正好服务于“让机器人拥有更好性能”的目标：不仅学会成功动作，也学会从偏差和危险状态中回来。

## 当前链路的输入输出总结

| 阶段 | 输入 | 输出 |
|---|---|---|
| 配置加载 | config `.py` | `TrainConfig` |
| norm stats | repo list、parquet、state/action | `assets/.../norm_stats.json` |
| Dataset 初始化 | LeRobot 数据、preprocessors | 有效 frame、权重、segment、prompt |
| DataLoader | Dataset、batch size、shuffle | batch: `Observation, Actions, ActionsMask` |
| Transform | 原始字段、norm stats、prompt | 模型输入格式 |
| 训练 | batch、pi05_base 权重 | loss、更新后的参数 |
| 保存 | TrainState | checkpoint step 目录 |

## 后续推理/评估入口

训练完成后，通常会用 policy server 或离线评估。

服务模型的通用入口：

```bash
uv run scripts/serve_policy.py policy:checkpoint \
  --policy.config=src/openpi/configs/cfg_pi0.5_seatbelt_14_dim.py \
  --policy.dir=checkpoints/seatbelt_14_dim_0401/seatbelt_14_dim_0401_exp/<step>
```

离线评估相关程序分散在：

```bash
tools/
scripts/benchmark/
src/openpi/configs/cfg_eval/
```

之前已验证过 openloop/baseline 相关测试能跑通，但真实部署 checkpoint 在单 L20 上恢复时出现过显存不足。因此训练、评估、部署三条链路后面需要分别确认显存预算。
