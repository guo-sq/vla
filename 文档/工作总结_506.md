## 4. 数据集概念与当前划分

本项目里 `repo` 不是 git 仓库，而是一个数据集文件夹。

数据根目录：

`/mnt/oss_data/anyverse_human_data_record/arxx5_bimanual/seatbelt`

一个 repo 的例子：

`seatbelt.single.avoid_obstacle_0_move.baichenglong.20260225.batch.10`

可拆解为：

- `seatbelt`：安全带任务。
- `single`：单侧/单任务设置。
- `avoid_obstacle_0_move`：具体任务类型。
- `baichenglong`：采集人/操作者。
- `20260225`：采集日期。
- `batch.10`：第 10 批数据。

repo 内部结构通常包含：

- `data/chunk-000/episode_*.parquet`：每个 episode 的结构化数据。
- `videos/chunk-000/`：对应视频。
- `meta/`：数据集元信息。
- `.hf_arrow_cache/`：读取缓存。

核心层级：

`repo -> episode -> frame -> action`

含义：

- `repo`：一批同类示教数据。
- `episode`：一次完整示教。
- `frame`：示教中的一个时间点。
- `action`：该时间点机器人执行的控制量。
- `action chunk`：模型一次预测的未来一段动作，本项目常用 `action_horizon=50`，30 FPS 下约 1.67 秒。

baseline v1 固定训练集：

- 15 个 repo。
- 覆盖 `hang`、`insert_move`、`take_off_move`、`avoid_obstacle`。

baseline v1 固定测试集：

- `test_seen_task` 4 个 repo：`hang`、`insert_move`、`take_off_move`、`avoid_obstacle` 的未训练 batch。
- `test_unseen_task` 4 个 repo：`fix_insert_move`、`avoid_collision`、`collision_recovery`、`back_home`。

新增能力：

- `--duration_analysis`
- `--duration_abs_tol`
- `--duration_rel_tol`
- `--duration_stable_steps`
- `--duration_dims`
- `--lazy_load_eval`

输出文件：

- `duration_summary.json`
- `duration_per_chunk.csv`

评测逻辑：

- 示教动作估计耗时
- 模型输出动作估计耗时
- 平均 speedup factor
- faster / same / slower chunk rate

该评测不是闭环真机速度，但适合作为 open-loop 阶段的加速指标。

## 7. 动作加速方向调研结论

本轮重点关注的是“机器人实际执行动作更快”，而不是单纯“模型推理更快”。

结论：

- pi0.6 / RECAP 涉及速度提升，但更偏 RL 后训练：通过 value function、advantage conditioning 和人工干预数据提升成功率与完成速度。
- pi0.7 更直接对应动作加速：它把 overall speed、quality、mistake 等元数据作为训练条件，推理时可以要求高速输出。
- DemoSpeedup 与当前目标高度相关：它通过熵/不确定性判断哪些示教片段可以更安全地加速，并生成加速后的 demonstrations。
- SAIL、Realtime-VLA V2、VLASH 更偏真实部署侧，包括速度调制、异步推理、future-state-aware 控制、动作调度等。
- FAST、PD-VLA、RTC、Adaptive Action Chunking 等方法更偏模型推理效率、实时控制和 chunk 调度，对线上系统也重要，但不是当前第一优先级。

因此，下一阶段最适合先做：

**pi0.7-lite 风格 speed-conditioned LoRA：自动统计 episode 速度，给 seatbelt 数据加速度条件，训练时学习快/慢动作差异，推理时固定要求 fast。**

严格复现 pi0.7 的标签体系工作量较大，因为它还包含 quality、mistake、subgoal images 和更丰富的 metadata。但当前可以先做最小可行版：只自动生成 `fast / normal / slow` 速度标签，不做人工逐帧标注。
