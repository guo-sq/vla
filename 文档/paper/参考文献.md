

**4. 和我们工作相关的论文**
我建议按学习路径读，不要一上来就啃所有 VLA。

第一组：理解我们现在训练的模型是什么。

- π0: A Vision-Language-Action Flow Model for General Robot Control  
  https://arxiv.org/abs/2410.24164  
  这是 OpenPI/pi0 系的核心背景，理解“图像+语言+状态 -> action chunk”的模型形式。

- π0.5: a Vision-Language-Action Model with Open-World Generalization  
  https://arxiv.org/abs/2504.16054  
  我们当前用的是 pi0.5 base，这篇要读。

- OpenVLA: An Open-Source Vision-Language-Action Model  
  https://arxiv.org/abs/2406.09246  
  理解 VLA fine-tuning 的通用范式。

第二组：理解 action chunk / 模仿学习。

- Action Chunking Transformer, ACT  
  关键词搜：`Action Chunking Transformer imitation learning robotics`  
  这是“预测未来一段动作”这种训练方式的重要背景。

- Diffusion Policy: Visuomotor Policy Learning via Action Diffusion  
  https://arxiv.org/abs/2303.04137  
  不是 VLA，但对理解“从视觉到动作轨迹”的策略学习非常有帮助。

- Octo: An Open-Source Generalist Robot Policy  
  https://arxiv.org/abs/2405.12213  
  学习通用机器人策略如何处理多数据、多任务、多机器人。

第三组：和我们“缩短任务时间、动作更快更稳”直接相关。

- DemoSpeedup: Accelerating Visuomotor Policies via Entropy-Guided Demonstration Acceleration  
  https://arxiv.org/abs/2506.05064  
  和我们非常相关：怎么把慢示教变成更快的训练数据。

- SAIL: Faster-than-Demonstration Execution of Imitation Learning Policies  
  https://arxiv.org/abs/2506.11948  
  和“比人类示教更快执行”直接相关。

- Real-Time Robot Execution with Masked Action Chunking  
  https://arxiv.org/abs/2601.20130  
  和 action chunk 在真实机器人上的实时执行有关。

- VLASH  
  https://arxiv.org/abs/2512.01031  
  关注异步推理、future state，对机器人部署端有价值。

第四组：理解大规模机器人数据训练。

- RT-1: Robotics Transformer for Real-World Control at Scale  
  https://arxiv.org/abs/2212.06817

- RT-2: Vision-Language-Action Models Transfer Web Knowledge to Robotic Control  
  https://arxiv.org/abs/2307.15818

- SmolVLA: Efficient Vision-Language-Action Model trained on LeRobot Community Data  
  https://huggingface.co/blog/smolvla

**推荐关键词**
你可以按这些搜：

```text
vision-language-action model robotics
VLA robot fine-tuning
OpenPI pi0.5 fine-tuning
robot imitation learning action chunking

```

**我的判断**
已有 seatbelt 模型是一个“能用的历史 baseline”，但它的数据处理比较粗：主要靠人工选 repo，没有看到系统的数据审计、静止段处理、关键片段加权。我们接下来最合理的研究路线就是：

```text
旧 seatbelt checkpoint 作为 baseline
-> 分析它当时用了什么数据
-> 对当前 281 个 repo 做系统审计
-> 做静止帧裁剪、insert/recover 加权、低质数据降权
-> 训练新模型
-> 和旧 checkpoint 做离线/实机对比
```

这和 DemoSpeedup、SAIL、RTC/action chunking 这些论文是同一个问题方向：**不是单纯训一个模型，而是让机器人在保持成功率的前提下执行得更快、更稳。**
