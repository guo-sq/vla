# Speedup 与 Seatbelt 模型加速性能评测记录

评测日期：2026-05-06

## 评测目标

对比三类动作序列在 seatbelt 测试集上的表现：

- 示教数据：测试集 ground truth action。
- speedup 模型输出：`cfg_pi0.5_28_dim.all_public_datasets_anyverse_speed_up_20260324/40000`。
- seatbelt 模型输出：`pi05_base_seatbelt_data_0205_0312_horizon_50_trtc_single_self_play_recovery/84999`。

核心指标：

- `MSE / MAE`：模型输出动作和示教动作的误差。
- `pred_mean_seconds` 与 `gt_mean_seconds`：模型动作 chunk 与示教动作 chunk 到达终点动作的估计耗时。
- `speedup_factor_gt_over_pred`：`gt_mean_seconds / pred_mean_seconds`，大于 1 表示模型动作比示教更快。
- `faster/same/slower chunk rate`：模型动作相对示教更快、相同、更慢的 chunk 比例。

动作耗时估计方式：对每个 50 帧 action chunk，计算序列首次进入自身最终动作附近并连续稳定 `stable_steps=3` 帧的位置。容差为 `max(abs_tol=0.02, rel_tol=0.05 * path_length)`；耗时分析维度使用双臂关节非夹爪维度 `0-5,7-12`。

## 本次运行结论

当前机器是单张 NVIDIA L20 46GB。seatbelt pi0.5 checkpoint 在 `restore_state` 阶段已经显存不足，尚未进入推理；speedup checkpoint 的原始配置文件与当前代码不兼容，配置加载阶段失败。因此，本次没有完成“两个当前 checkpoint 在多个测试集上完整跑分”的矩阵评测。

已额外使用历史产物补算了 speedup 模型在 seatbelt `hang.batch.5` 上的动作耗时指标。该历史结果显示：speedup 模型在该测试集上并没有整体快于示教，平均耗时略慢。

## 失败原因

### seatbelt 模型

尝试命令使用 `batch_size=4`、`sample_steps=4`、`lazy_load_eval=1`，数据已进入 lazy load，但在 checkpoint restore 阶段失败：

```text
jaxlib.xla_extension.XlaRuntimeError: RESOURCE_EXHAUSTED:
Out of memory while trying to allocate 4831838208 bytes.
```

这说明问题不在 batch size 或测试集，而是模型权重恢复到单张 46GB GPU 时已经超过可用显存。

### speedup 模型

尝试加载 `/mnt/oss_models/.../cfg_pi0.5_28_dim.all_public_datasets_anyverse_speed_up.py` 时失败：

```text
TypeError: TrainConfig.__init__() got an unexpected keyword argument 'window_datasets'
```

这说明该 speedup checkpoint 对应的训练代码版本比当前本地 `openpi_modified` 更新，当前 `TrainConfig` 缺少 `window_datasets` 字段。

## 本次为评测补充的脚本能力

已在 `openpi_modified/scripts/test.py` 中加入：

- `--duration_analysis`：输出动作完成时间对比。
- `--duration_dims`：指定耗时分析动作维度，例如 `0-5,7-12`。
- `--duration_abs_tol`、`--duration_rel_tol`、`--duration_stable_steps`：控制动作完成判定。
- `--lazy_load_eval`：评测时启用 episode 级 parquet lazy load，避免整库生成 HF Arrow cache。

## 建议

要完成真实对比，优先处理两件事：

1. 在多卡或更大显存机器上运行 seatbelt 模型；当前单卡 46GB 在恢复权重阶段就不足。
2. 将本地 `openpi_modified` 与 speedup checkpoint 对应的训练代码版本对齐，至少补齐 `TrainConfig.window_datasets` 及其下游逻辑。

完成上述两点后，再固定同一批测试集，例如：

- `seatbelt.single.avoid_obstacle_0_move.baichenglong.20260225.batch.10`
- `seatbelt.single.avoid_collision_1_move.zhaoshuai.20260306.batch.7`
- `seatbelt.single.hang.baichenglong.20260205.batch.5`

即可得到 seatbelt 模型、speedup 模型和示教数据三者的完整横向表格。
